### Example 1: Ask Cooper to Build a Feature

**Your request:**
```
Cooper, build user authentication with OAuth, JWT, and email verification
```

**What happens automatically:**

1. **Cooper → Sage** (with @architect, @api-design, @postgres-patterns)
   ```
   Sage designs:
   - OAuth flow architecture
   - API contracts (POST /auth/login, /auth/callback, /auth/verify)
   - User/Session/Token data models
   - ADR documenting OAuth provider choice
   - Pre-mortem analysis
   ```

2. **Cooper → Forge** (with @tdd-workflow, @verification-loop, @python-patterns)
   ```
   Forge implements:
   - Writes failing tests FIRST (RED)
   - Implements OAuth endpoints (GREEN)
   - Runs verification loop:
     ✓ Build passes
     ✓ Types check
     ✓ Lint passes
     ✓ Tests pass (15/15)
     ✓ Coverage: 84% (target: 80%+)
   ```

3. **Cooper → Vigil** (with @security-scan, @code-review, @security-review)
   ```
   Vigil reviews:
   - Runs @security-scan
     ✓ No hardcoded secrets
     ✓ SQL injection prevention (parameterized queries)
     ✓ CSRF protection enabled
     ✓ Rate limiting on auth endpoints
     ✓ Token expiry configured
   - Applies @code-review
     ✓ Functions < 50 lines
     ✓ Error handling present
     ✓ No deep nesting
   - Rating: 5/5 (Exceptional)
   - Decision: APPROVED ✅
   ```

4. **Cooper → You**
   ```
   ✅ Authentication system implemented:
   - OAuth with Google/GitHub
   - JWT tokens with refresh
   - Email verification flow
   - 84% test coverage
   - Security reviewed and approved
   - Ready to deploy
   ```
